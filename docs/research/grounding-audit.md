## exec-ssh

_Verify:_ I re-verified all 11 claims in the exec-ssh grounding audit by opening every cited emdash, ACP, happy-cli, hapi, and nimbalyst source file directly (and the two vendored/external doc references: ssh2's README.md and minisign/sigstore/systemd docs via WebFetch). Two things need a second look before this is folded back into the spec's grounding notes:

1. Claim 7's recommendation slightly overstates how 'out of scope' the dependency/install-command source is: packages/core/src/host-dependencies/core-dependencies.ts and packages/plugins/src/agents are present in the same emdash checkout (not a separate unavailable package) — I read core-dependencies.ts and confirmed it only defines install commands for git (brew/apt/winget), with no mise-equivalent bootstrap recipe anywhere in packages/core or packages/plugins. The end conclusion (no install-recipe precedent exists) survives, but the framing that these files 'live outside this checkout scope' is inaccurate and should be corrected.

2. Claim 10 ('runs headless via systemd' is ungrounded, only happy-cli's macOS-only launchd installer exists as a weak analogue) is REFUTED as written. The audit's own grep evidence claims 'only unrelated hits' turned up when searching for 'systemd'/'.service' across the reference set — but hapi/docs/guide/installation.md, one of the five reference sources the audit was scoped to, contains a complete, directly-applicable systemd user-unit precedent: hand-authored `hapi-hub.service` and `hapi-runner.service` unit files (Type=simple, Restart=always, RestartSec=5, WantedBy=default.target) plus systemctl enable/start instructions and a documented KillMode=process rationale for not killing detached agent sessions on runner restart — exactly the resident-supervisor-on-Linux pattern loombox needs. This should be cited as real evidence (partial-to-grounded), not waved off as ungrounded with only a macOS analogue offered.

Net: the audit is high-quality and mostly trustworthy, but it should be corrected on these two points, most importantly point 2, before being treated as final.</summary>
